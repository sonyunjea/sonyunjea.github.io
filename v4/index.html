프로젝트 개요

본 보고서는 스마트 우산꽂이 시스템의 소프트웨어 개발 전 과정을 요약합니다. 단일 언어(JavaScript)를 사용하여 Node.js/Express 기반의 백엔드 서버와 React 기반의 프론트엔드 앱을 개발했으며, Capacitor를 통해 안드로이드/iOS 네이티브 앱으로 최종 패키징했습니다.

시스템 아키텍처

시스템은 각 요소의 독립성을 보장하는 3계층 아키텍처를 채택했습니다. 모든 데이터는 중앙 서버의 REST API를 통해 교환됩니다.

기술 스택

프로젝트의 신속한 개발과 확장성을 위해 검증된 최신 기술들을 조합하여 사용했습니다. 전체 기술 스택은 프론트엔드, 백엔드, 그리고 배포/빌드 영역으로 구성됩니다.

기술 스택 분포

본 프로젝트는 JavaScript 생태계를 중심으로 구성되어 있으며, 각 영역별로 전문화된 라이브러리와 프레임워크를 활용했습니다. 아래 차트는 각 영역이 전체 프로젝트에서 차지하는 기술적 중요도를 시각적으로 보여줍니다.

백엔드 아키텍처

Node.js와 Express를 사용하여 '상태 저장'과 '명령 중계'라는 두 가지 핵심 기능에 집중하는 경량 API 서버를 구축했습니다. 별도의 데이터베이스 없이 인메모리 방식으로 상태를 관리하여 신속한 개발을 추구했습니다.

API 엔드포인트

GET /api/device/status

서버에 저장된 현재 기기 상태를 JSON으로 조회합니다.

POST /api/device/control

클라이언트의 요청에 따라 서버의 기기 상태를 변경합니다.

API 역할 비교

두 API는 명확히 다른 역할을 수행합니다. `GET` 요청은 단순히 정보를 조회(읽기)하는 반면, `POST` 요청은 서버의 상태를 실제로 변경(쓰기)하는 역할을 담당합니다. 이 차이는 시스템 안정성의 핵심입니다.

프론트엔드 & UI/UX

사용자 경험(UX) 향상을 위해 직관적이고 반응성이 뛰어난 사용자 인터페이스(UI)를 구축하는 데 중점을 두었습니다. Tailwind CSS로 스타일링하고, framer-motion으로 애니메이션 효과를 적용했습니다.

UI/UX 구현 요소

좋은 앱 경험은 여러 요소의 조합으로 만들어집니다. 본 프로젝트에서는 기능별 컴포넌트화, 사용자 피드백, 그리고 시각적 효과에 자원을 배분하여 개발했습니다.

주요 UI 컴포넌트

  • WeatherCard & HourlyForecast: 외부 API(OpenWeatherMap) 데이터를 시각화하여 날씨 정보를 제공.
  • FanControl & UVCControl: 사용자가 시간, 세기 등을 설정할 수 있는 인터랙티브 제어판.
  • DeviceStatusCard: 백엔드로부터 받은 기기 상태를 종합적으로 보여주는 대시보드.
  • 피드백 요소: 로딩 스피너, 오류 알림 등을 통해 사용자에게 시스템의 현재 상태를 명확히 전달.

빌드 & 배포

React 웹 프로젝트를 네이티브 앱으로 빌드하고, 백엔드 서버를 클라우드에 배포하여 언제 어디서든 접근 가능한 시스템을 구축했습니다.

자동화된 배포 파이프라인 (CI/CD)

GitHub와 Render.com을 연동하여, 코드 변경사항이 `git push`를 통해 깃허브에 올라가면 자동으로 새로운 버전의 서버가 배포되도록 CI/CD 파이프라인을 구축했습니다.

프론트엔드 빌드

완성된 React 웹 프로젝트에 Capacitor를 적용하여 `android`와 `ios` 네이티브 프로젝트를 생성했습니다. `npm run build`와 `npx cap sync` 명령어를 통해 단일 코드베이스의 변경사항을 양 플랫폼에 효율적으로 반영하는 워크플로우를 구축했습니다.

백엔드 배포

백엔드 서버는 Render.com에 배포되었으며, GitHub 저장소와 연동하여 `git push` 시 자동으로 서버가 재배포되는 CI/CD 파이프라인을 통해 운영 효율성을 확보했습니다. 이를 통해 개발자는 코드 수정에만 집중할 수 있습니다.
